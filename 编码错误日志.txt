一、main.dart 修复 (lib/main.dart)

变更后内容:

  await storageService.init();
  
  runApp(...);

修复说明:
- 添加了 WidgetsFlutterBinding.ensureInitialized() 调用

- 解决移动端调用SharedPreferences时的白屏/闪退问题

三、ssh_direct_bridge.dart 修复 (lib/services/ssh_direct_bridge.dart)
----------------------------------------------------------------
修复内容:
1. UDP端口复用修复:
   变更前: _udpSocket = await RawDatagramSocket.bind(InternetAddress.anyIPv4, 9999);
   变更后: _udpSocket = await RawDatagramSocket.bind(InternetAddress.anyIPv4, 9999, reuseAddress: true);
   修复说明: 添加reuseAddress: true参数，防止移动端切屏导致端口占用报错

2. Python脚本IPv6支持修复:
   变更前: 
   sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
   TARGET = (phone_ip, 9999)
   
   变更后:
   # 【修复】动态识别 IP 协议族。手机连局域网常被分配 IPv6 映射地址
   # 若写死 AF_INET，会导致 Python 侧直接抛出异常崩溃，导致手机端再也收不到数据（黑屏）
   try:
       if ':' in phone_ip:
           sock = socket.socket(socket.AF_INET6, socket.SOCK_DGRAM)
       else:
           sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
       TARGET = (phone_ip, 9999)
   except Exception:
       sys.exit(1)
   
   修复说明: 动态识别IP协议族，支持IPv6地址，防止Python侧崩溃导致手机端收不到数据

3. HttpClient泄漏修复:
   变更前:
   Future<void> _fetchMap(String ip) async {
     try {
       final client = HttpClient()..connectionTimeout = const Duration(seconds: 3);
       ...
     } catch (e) {}
   }
   
   变更后:
   Future<void> _fetchMap(String ip) async {
     HttpClient? client;
     try {
       client = HttpClient()..connectionTimeout = const Duration(seconds: 3);
       ...
     } catch (e) {
       // 网络波动时忽略
     } finally {
       // 【关键修复】移动端连接池资源极度有限(不像Linux有上万个fd)。
       // 每4秒创建一次Client如果不主动释放，1分钟内导致整机网络阻塞断开连接，出现黑屏失控。
       client?.close(force: true); 
     }
   }
   
   修复说明: 确保HttpClient被正确关闭，防止移动端连接池资源耗尽导致网络阻塞
